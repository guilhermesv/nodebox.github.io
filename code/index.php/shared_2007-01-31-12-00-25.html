Posted by Tom De Smedt on Feb 1, 2007

Hi Chris,

Python as a programming language is a good all-rounder: it's understandable for people without any coding experience, it can do OO as well as functional programming and has lots of libraries and functionalities. Naturally, the same can be said about Perl and Ruby. In the past we have even thought about developing a version for Ruby.

However, Frederik and me are Python programmers with no real knowledge of Perl. So a Perl version is not something we are currently thinking about. It would involve developing in PerlObjC and rewriting all of the code. NodeBox is basically a way to have Python talk to the Quartz technology in Mac OS X. A Perl interface would be a totally new, totally different application.
